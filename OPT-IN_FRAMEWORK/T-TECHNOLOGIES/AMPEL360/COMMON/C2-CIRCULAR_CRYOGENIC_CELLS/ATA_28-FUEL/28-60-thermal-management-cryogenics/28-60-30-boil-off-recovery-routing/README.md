# ATA 28-60-30 — Boil-Off Recovery and Routing

| Field | Value |
|---|---|
| **PCT** | PCT-28-BWB |
| **Status** | ACTIVE |

## Scope

Capture and routing of LH₂ boil-off vapour generated by parasitic heat leak
through the tank insulation. Boil-off is **recovered**, not jettisoned:

1. **Primary path** — route to conversion-stack inlet for productive use
2. **Secondary path** — feed the vapor-cooled-shield (VCS) to reduce
   subsequent boil-off (closed-loop recovery)
3. **Tertiary path** — controlled overboard vent **only when** primary and
   secondary are saturated, with thermal-plume management

### Why this is not 28-30 (Dump / Jettison)

A jettison function discards mass to achieve a defined safe state
(e.g. weight reduction for emergency landing). Boil-off recovery is the
opposite: it is a continuous, normal-operations function whose objective is
to **conserve** PM mass. Vent-as-last-resort is a fault-tolerance fallback,
not the primary intent. Cataloguing recovery under 28-30 conflates these two
classes and misleads downstream MSG-3 and FHA analyses.
