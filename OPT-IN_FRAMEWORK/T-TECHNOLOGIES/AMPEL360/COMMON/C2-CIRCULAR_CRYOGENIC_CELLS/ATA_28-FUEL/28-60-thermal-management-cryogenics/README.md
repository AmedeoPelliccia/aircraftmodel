# ATA 28-60 — Thermal Management & Cryogenics

| Field | Value |
|---|---|
| **ATA-100 Slot** | 28-60 Reserved (used as ATA-extension) |
| **PCT** | PCT-28-BWB |
| **BREX Extension** | Declared in BREX-IDA360-Q100-v0.1 — extension justification: cryogenic thermal management has no analogue in legacy ATA-28 |
| **Status** | ACTIVE |

## Scope

Cryogenic thermal-management functions specific to the LH₂ path. Use of the
reserved slot is **explicitly declared** in the project BREX with extension
justification.

## Subsections

| Slot | Title | Notes |
|---|---|---|
| `28-60-10-vacuum-jacket-monitoring/` | Vacuum integrity, MLI degradation | |
| `28-60-20-pre-cool-and-conditioning/` | Pre-cool of feed lines and PEMFC inlet | Couples with `28-20-30/` if NH₃ |
| `28-60-30-boil-off-recovery-routing/` | Vent / recycle of boil-off vapour | **Relocated** from 28-30 (semantic correction: recovery ≠ jettison) |

## Rationale for boil-off relocation

Boil-off recovery is a **closed-loop thermal-management** function: vapour
generated by parasitic heat leak is captured and either (a) routed back to
the conversion stack, (b) used for vapor-cooled-shield pre-cool, or
(c) vented in a controlled manner only when (a) and (b) are saturated.
None of these is a jettison (controlled discard) function. Placing it under
28-30 conflated *recovery* with *discard* and broke ATA-100 semantics. The
correct home is 28-60.
